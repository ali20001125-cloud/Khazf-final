# الدفعة ١ — الأساس وقاعدة البيانات ✓

## ما أُنجز
**م١ التأسيس:**
- حُذف `output: "export"` (SSR فعّال — شرط المصادقة والطلبات)
- Drizzle ORM + pg + tsx مثبّتة · `.env` / `.env.example` جاهزان
- **قرار بيئة:** Prisma محجوب في بيئة البناء (محركاته تُحمَّل من binaries.prisma.sh → 403).
  طُبّقت قاعدة الوثائق «خذ المنطق وترجم التفاصيل التقنية» → **Drizzle**:
  نفس النواتج (Models/Relations/Indexes/Migrations/Transactions) وهجرات SQL خام
  — وهي الصيغة نفسها التي يستقبلها محرر Supabase.

**م٢ قاعدة البيانات (٢١ جدولاً):**
- المنتجات (قهوة/أدوات، كل الأسعار يدوية، أماكن الظهور بدل التصنيفات، سلوك النفاد لكل منتج)
- المخزون: **وجبات FIFO** (التكلفة مع الوجبة فقط) + **سجل الحركات** — الرصيد محسوب، لا حقل مخزون على المنتج
- الطلبات (تسلسل KHZ-1001+، لقطة الحساب الكامل: خصومات/نقاط/توصيل زبون-شركة-صافي/ربح/تقريب) + الأسطر
- العملاء (الهاتف = المعرّف) + المفضلة
- الولاء: سجل الكاش باك (متاح بعد ٤٨ سا) + رحلة ٦ مستويات قابلة للتعديل + هدايا البوكس
- الأكواد العامة + استخداماتها (حدود كلية/لكل زبون/موجّهة)
- التقييمات (موافقة إدارية + شراء موثّق) · البنرات · الإعدادات (عامة/داخلية منفصلتان)
- `sql/supabase-rls.sql`: أمان صفوف كامل (is_admin + jwt_phone، قراءة عامة للكتالوج، العميل يرى بياناته فقط)

**مُختبَر فعلياً على PostgreSQL 16 محلي:**
- الهجرات ✓ · الزرع الحقيقي ✓ (بيانات DATA-CORRECTIONS حصراً — بلا قصص/تقييمات مخترعة، مخزون صفر)
- بيع FIFO داخل Transaction بقفل صفوف `FOR UPDATE` + تراجع كامل ✓ (٥٠٠٠→٤٧٥٠→٥٠٠٠)

**تعديلات الواجهات العاجلة:**
- KHAZF10 محذوف كلياً (توب بار/سلة/عروض/store) · صفحة العروض أعيدت (بوكس + ولاء، بلا بوكسات جاهزة)
- التوصيل ٣٬٠٠٠ د.ع في السلة والدفع (lib/config مؤقت حتى م٥)
- بيانات كالدي/سيرادو/الدورادو مصحّحة · قصة المحصول تظهر فقط إن كتبها المالك

## الأوامر
`npm run db:generate` · `db:migrate` · `seed` (يدوي) · `seed:demo-stock` (تجريبي) · `db:verify`
